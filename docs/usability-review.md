## 검토 범위와 방법

- 프론트엔드 전체: 홈/워크스페이스(`HomeView`/`WorkspaceView`/`ChatPanel`/`AssistantMessage`/`CanvasPanel`/`FileViewer`/`RequirementsForm`), 셸(`AppShell`/`TopBar`/`NavRail`), 설정 3뷰(`AgentsView`+`AgentCard`/`FlowSettingsView`/`KnowledgeView`), 가이드 플로우 라이브러리(`lib/workflow.ts`/`options.ts`/`skills.ts`/`clarify.ts`).
- 백엔드는 사용자 노출 메시지 표면만: `rag.rs`/`confluence.rs`의 에러 문구, `types.ts`의 진단 힌트.
- 관점: 진행 피드백 / 에러 복구 / 온보딩·빈 상태 / 입력 UX / 내비게이션 / 워크플로우 가시성 / 취소·정지 / 파괴적 동작 보호 / 일관성(언어 포함) / 편의 기능.

## 총평

전반적으로 **핵심 플로우의 골격은 잘 잡혀 있다.** 특히 다음은 이미 좋은 수준이다:

- 요구사항 폼 대기 중 채팅 차단의 **이유 안내**(placeholder "오른쪽 '요구사항' 항목을 먼저 작성해 제출해 주세요" + 버튼 tooltip — `ChatPanel.tsx:986-992`, `:1008`).
- 턴 실패 시 **한글 힌트 + "새 세션으로 다시 시도"** 액션(`AssistantMessage.tsx:111-126`, `workspace.ts:75-90`).
- 채팅/캔버스 **드래그 리사이즈 + 폭 기억**(`WorkspaceView.tsx:189-214`), 리사이즈 중 iframe 이벤트 실드까지 처리(`:290-292`).
- Confluence 수집의 **화면 이탈 생존**(모듈 스토어 `lib/ingest.ts`) + NavRail pulse 점(`NavRail.tsx:38-40`).
- Flows 편집기의 **인라인 검증**(마지막 단계=대화 등)과 pinned 기반 단계의 tooltip 설명, 스킬 사용처 역참조 배지.
- 지식 뷰의 설명 문단(평문 저장 경고, RAG 스텁 안내 포함)과 빈 상태 문구.

반면, **긴 자동 워크플로우의 진행 가시성**, **에러 후 복구 경로**, **처음 사용자의 미탐지 온보딩**, **파괴적 동작 보호**, **언어 혼용** 다섯 축에서 체감 사용성을 깎는 공백이 뚜렷하다. 아래 우선순위로 정리한다.

## P0 — 체감 효과가 가장 큰 5건

### 1. 워크플로우 진행 스테퍼 부재 (가시성)

- **현상**: plan 기본 8단계(기반 3단계 + 문서 4종 + 대화)가 자동 연쇄되는 동안, 진행 표시는 채팅에 흘러가는 일시적 시스템 노트("N/M단계 · <이름> 중…" — `workflow.ts:303-306`, 렌더 `ChatPanel.tsx:923-929`)뿐이다. 스크롤되면 사라지고, "지금 몇 단계인지 / 앞으로 뭐가 남았는지"를 한눈에 보는 지속적 UI가 없다. 단계 건너뜀(RAG 미설정 등)도 시스템 노트 한 줄로만 지나간다(`ChatPanel.tsx:449`, `:455`, `:459`, `:467`, `:471`). 요구사항 폼 단계에서도 제출 후 어떤 단계들이 실행될지 미리 보여주지 않는다.
- **개선 방향**: 워크스페이스(채팅 헤더 아래 또는 캔버스 툴바)에 **지속적 스테퍼/체크리스트**를 표시 — 완료(✓)/진행 중(스피너)/건너뜀(－)/대기 상태, 현재 단계 하이라이트. 요구사항 폼 하단에 "제출 시 진행될 단계" 미리보기 추가. 단계 커서(`stepIndexRef`)는 이미 있으므로 상태를 `WorkspaceView`로 리프트하면 된다.
- **변경 범위**: 중 (ChatPanel 커서 상태 리프트 + 신규 스테퍼 컴포넌트, 백엔드 무변경).

### 2. 채팅 응답이 마크다운으로 렌더되지 않음

- **현상**: assistant 응답은 `whitespace-pre-wrap` 평문으로 표시된다(`AssistantMessage.tsx:103-107`). 에이전트가 내는 표/코드블록/제목/링크가 원시 마크다운 텍스트로 노출된다. 마크다운 렌더러(`MarkdownView`, mermaid 포함)는 이미 존재하지만 **파일 뷰어에서만** 쓰인다(`FileViewer.tsx:124`, `Markdown.tsx:68`).
- **개선 방향**: `AssistantMessage`의 본문을 `MarkdownView`로 교체(스트리밍 중 부분 마크다운 렌더 깨짐은 완료 후 렌더 전환 또는 관대한 파싱으로 완화). 코드블록에는 하이라이팅까지는 아니어도 mono 블록 + 복사 버튼(P1-10과 연계).
- **변경 범위**: 소~중 (기존 컴포넌트 재사용; 채팅 폭에 맞는 `.markdown-body` 스타일 조정).

### 3. 에이전트 미탐지 온보딩 부재

- **현상**: 탐지된 에이전트가 0개여도 홈/워크스페이스에 아무 안내가 없다. 컴포저 드롭다운은 미탐지 에이전트를 "(미탐지)" 접미사로 나열하고 선택·전송을 막지 않는다(`ChatPanel.tsx:953-958`) — 전송해야 런타임 에러로 실패를 발견한다. "Agents 화면에서 경로를 설정하세요"로 유도하는 빈 상태/링크가 없고, 부팅 직후 탐지 진행 중이라는 표시도 없다(`App.tsx:50-68`는 셸을 즉시 렌더하고 백그라운드 탐지).
- **개선 방향**: ① 탐지 결과 0개면 홈 히어로 아래에 안내 배너("사용 가능한 에이전트가 없습니다 — Agents에서 경로를 설정하세요" + 이동 버튼). ② 미탐지 에이전트 선택 시 컴포저 위에 경고 라인 + Agents 이동 링크. ③ 초기 탐지 진행 중에는 드롭다운에 "탐지 중…" 표시.
- **변경 범위**: 소 (배너/경고 라인 추가; `detected` 상태는 이미 App이 보유).

### 4. 진행 중 이탈 시 무경고 취소 + 정지 피드백 불일치

- **현상**: 홈 이동/새 세션/기록 열기 시 `ChatPanel` 언마운트 훅이 실행 중 런을 **조용히 취소**한다(`ChatPanel.tsx:220-224`) — 문서 4종을 생성하는 긴 자동 연쇄 도중에도 확인 없이 끊긴다. 또 Stop의 사후 안내가 경로마다 다르다: preflight fetch 중 정지는 "단계 진행을 중지했습니다." 노트가 남지만(`ChatPanel.tsx:609`), 에이전트 스트림 중 정지는 자동 진행만 조용히 중단되고(`:401-405`) "중지됨" 노트가 없다 — 정지 후 워크플로우가 일반 대화로 강등된다는 사실을 사용자가 알 수 없다.
- **개선 방향**: ① 스트리밍/자동 연쇄 중 이탈 액션에 확인 다이얼로그("진행 중인 작업이 중단됩니다"). ② 모든 정지 경로에 동일한 시스템 노트("작업을 중지했습니다 — 이후 일반 대화로 진행됩니다") 추가.
- **변경 범위**: 소 (노트 1줄 + confirm 게이트; `streaming` 상태는 이미 리프트됨).

### 5. 같은 세션 내 재시도(retry) 부재

- **현상**: 턴 실패의 유일한 복구 액션이 "새 세션으로 다시 시도"다(`AssistantMessage.tsx:115-124`). 스트림이 중간에 끊기면 그때까지의 대화 맥락을 유지한 채 마지막 턴만 재전송할 방법이 없다 — 세션형(claude/codex)은 `--resume`으로 재시도가 충분히 가능함에도 새 세션으로 맥락을 버리게 유도한다.
- **개선 방향**: 에러 박스에 **"다시 시도"**(마지막 사용자 턴 재전송, 같은 세션 resume) 버튼을 1차 액션으로 추가하고 "새 세션으로 다시 시도"는 2차 액션으로 유지. 마지막 wire/프롬프트는 ref로 보관하면 된다.
- **변경 범위**: 소~중 (마지막 턴 보관 + send 재호출 경로; 워크플로우 커서 되감기 규칙만 주의).

## P1 — 신뢰감·안전성

### 6. RAG/지식 preflight 중 무표시

- **현상**: rag/knowledge 단계의 preflight는 `setStreaming(true)`로 Stop만 활성화하고(`ChatPanel.tsx:597-600`), 메시지 버블은 fetch가 끝난 **뒤**에야 추가된다(`:671`). 느린 사내 RAG 서버 호출(타임아웃 120초 — D53) 동안 화면에는 아무 표시가 없다.
- **개선 방향**: preflight 시작 시점에 시스템 노트("사내 문서 검색 중…" / "지식 베이스 확인 중…")를 즉시 추가.
- **변경 범위**: 소.

### 7. 파괴적 동작 확인 부재

- **현상**: 앱 전체에 확인 다이얼로그가 하나도 없다(`confirm`/Tauri ask 사용처 0). 특히 **지식 항목 삭제는 즉시 영속 + undo 없음**(`KnowledgeView.tsx:483-491`, `:453-462`). Flows "기본값으로"(커스텀 단계/스킬 전체 폐기 — `FlowSettingsView.tsx:441-449`, `:707-715`)와 Agents "Clear"(`AgentCard.tsx:126-136`)도 클릭 한 번에 실행된다.
- **개선 방향**: 즉시 영속되는 삭제/복원(지식 삭제, 기본값 복원, Clear)에 확인 단계 도입. draft 전용 삭제(Flows 단계/스킬)는 현행 유지해도 무방(저장 전 취소 가능).
- **변경 범위**: 소.

### 8. Flows 미저장 편집의 조용한 폐기

- **현상**: 단계/스킬 draft는 컴포넌트 로컬 state이고, 카테고리 탭 전환이 `key={category}` 리마운트로 미저장 편집을 조용히 버린다(`FlowSettingsView.tsx:771`). 뷰 이동도 동일. dirty 경고가 없다.
- **개선 방향**: dirty 상태에서 탭 전환/이탈 시 확인("저장하지 않은 변경이 있습니다") 또는 최소한 저장 버튼 옆 "변경됨" 배지.
- **변경 범위**: 소~중 (dirty 비교 + 탭 전환 게이트).

### 9. 무음 실패 3건

- **현상**: ① 홈 최근 프로젝트 열기 실패 시 안내 없이 새 빈 대화로 폴백(`HomeView.tsx:53-63`), ② 기록 팝오버의 세션 로드 실패 무음(`ChatPanel.tsx:755-757`), ③ 폴더 픽커 실패/취소 무음(`HomeView.tsx:38-41`, `RequirementsForm.tsx:28-31` — 취소는 무음이 맞지만 에러도 무음).
- **개선 방향**: 실패 시 토스트/인라인 안내("세션을 열지 못했습니다: <사유>"). App 부트 배너(`App.tsx:118-141`) 패턴 재사용 가능.
- **변경 범위**: 소.

### 10. 복사 버튼 전무

- **현상**: 코드 전체에 clipboard 사용처가 없다. 채팅 응답/코드블록/파일 경로/RAG 결과 어디에도 복사 버튼이 없다 — 산출물을 다른 곳에 옮기는 기본 동선이 막혀 있다.
- **개선 방향**: assistant 메시지 hover 복사 버튼 + (P0-2 마크다운 렌더 후) 코드블록 복사 버튼, 캔버스 폴더 칩/파일 탭에 경로 복사.
- **변경 범위**: 소.

### 11. 강제 자동 스크롤

- **현상**: 메시지 변경마다 무조건 하단으로 스크롤한다(`ChatPanel.tsx:247-249`) — 긴 스트리밍 중 위로 올려 과거를 읽는 동안에도 새 델타마다 하단으로 끌려간다. "맨 아래로" 버튼도 없다.
- **개선 방향**: 사용자가 하단 근처에 있을 때만 자동 스크롤(bottom-pinned 감지), 벗어나 있으면 "↓ 최신으로" 플로팅 버튼.
- **변경 범위**: 소.

## P2 — 일관성·완성도

### 12. 언어 혼용 (한글 통일 필요)

| 위치 | 현상 |
|------|------|
| Agents 뷰 전체 | 제목/부제/상태(`Detecting…`/`Detected`/`Not detected`)/버튼(`Refresh`, `Save & detect`, `Clear`, `Browse…`)/경로 설정 설명까지 전면 영어 (`AgentsView.tsx:26-32`, `AgentCard.tsx` 전반) |
| 탐지 진단 힌트 | `DIAGNOSTIC_HINT` 3종 모두 영어 문장 (`types.ts:231-236`) — 미탐지 카드에 그대로 노출 |
| NavRail | "Home/Agents/Flows"(영) + "지식"(한) 혼재 (`NavRail.tsx:8-12`) |
| 공통 피드백 | 저장 성공이 세 뷰 모두 영어 `Saved.` (`AgentCard.tsx:137`, `FlowSettingsView.tsx:439`, `KnowledgeView.tsx:173` 등) — 트리거 버튼은 Agents만 영어, Flows/지식은 한글 |
| 채팅 | "reasoning 보기/숨기기"(`AssistantMessage.tsx:93`), "{n} in · {n} out tokens"(`:55-58`), "(no output)"(`:51`) |

- **개선 방향**: 사용자 노출 문자열을 한글로 통일(1차 대상: Agents 뷰 + DIAGNOSTIC_HINT + NavRail + "Saved."). 문자열이 흩어져 있으므로 이번 기회에 상수 모듈로 모으는 것도 고려(본격 i18n 프레임워크는 과함).
- **변경 범위**: 중 (기계적이지만 파일 수가 많음).

### 13. RAG 미구현 스텁의 개발자용 메시지 노출

- **현상**: 지식 뷰의 RAG 폼은 끝까지 채우고 "연결 테스트"까지 누를 수 있지만, `rag.rs` 스텁이 미구현이면 항상 "RAG search 미구현 — src-tauri/src/rag.rs의 RagClient::search를 채워 주세요"(`rag.rs:125`)가 빨간 에러로 노출된다(`KnowledgeView.tsx:162-171`). 비개발자 사용자에게는 원인 파악이 불가능한 함정이다.
- **개선 방향**: 미구현 상태를 구분해 사용자용 문구로 표시("이 빌드에는 RAG 연동이 포함되지 않았습니다 — 관리자에게 문의"). 폼 상단에 비활성 배지로 선제 안내하면 더 좋다.
- **변경 범위**: 소.

### 14. 발화 주체가 항상 "Operation Wizard"

- **현상**: assistant 메시지 헤더가 실행 에이전트와 무관하게 "Operation Wizard" 고정이다(`AssistantMessage.tsx:83`). 세션마다 에이전트가 다를 수 있는데(새 세션 시 재선택) 대화 로그만 봐서는 어떤 에이전트/모델의 응답인지 알 수 없다.
- **개선 방향**: 헤더에 에이전트명(+선택 모델)을 표시하거나 최소한 세션 시작 시스템 노트로 남긴다.
- **변경 범위**: 소 (`ChatMessage`에 agentId가 이미 세션 메타로 존재).

### 15. 홈 컴포저의 카테고리 고정(plan)과 관계 불명확

- **현상**: 홈 컴포저의 Enter/전송은 항상 `plan` 카테고리로 시작한다(`HomeView.tsx:36`). 하단 칩도 "개발 계획 수립" 고정. 프롬프트를 쓴 뒤 카테고리 카드를 누르면 그 프롬프트가 해당 카테고리로 전달되지만 이 관계가 UI에 설명돼 있지 않다.
- **개선 방향**: 컴포저에 카테고리 선택(칩 4개 토글)을 넣거나, 카드에 "위 프롬프트로 시작" 힌트 표시.
- **변경 범위**: 소~중.

### 16. 기타 편의 공백

| 항목 | 현상/근거 | 제안 | 범위 |
|------|-----------|------|------|
| 다크모드 인앱 토글 없음 | 다크 토큰은 존재(`tokens.css:66`, `:106-108`)하나 `data-theme` setter가 코드에 없음 — OS 설정만 따름 | TopBar에 라이트/다크/자동 토글 | 소 |
